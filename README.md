# CS 405: Secure Coding Portfolio
**Author:** Caleb Sorell  
**Course:** CS 405 - Southern New Hampshire University  

## Overview
This repository serves as a comprehensive portfolio of my work in CS 405: Secure Coding. The artifacts included demonstrate my ability to analyze, design, and implement secure software practices, with a focus on shifting security left in the software development life cycle (SDLC).

## Portfolio Contents

### 1. Security Policy (Project One)
A comprehensive security policy designed for the "Green Pace" development team. This document outlines the transition to a DevSecOps environment, detailing mandated C++ coding standards, vulnerability mitigation strategies, and automated testing requirements.

### 2. Security Policy Presentation (Project Two)
A management-level slide deck presenting the core components of the Security Policy. 
* *Note: The video recording of this presentation has been submitted directly to the instructor via an unlisted YouTube link.*

### 3. C++ Secure Coding Artifacts
Zipped directories containing source code and unit tests developed throughout the course. These artifacts demonstrate practical implementation of secure coding standards, including:
* Input validation and sanitization
* Prevention of integer overflows/underflows
* Secure memory management
* Implementation of encryption and decryption logic

### 4. Portfolio Journal Reflection
A final reflective essay synthesizing the core concepts of the course, including the business value of DevSecOps, risk assessment methodologies, and the application of a Zero Trust architecture.

## Technologies and Concepts Demonstrated
* **Languages:** C++
* **Security Standards:** SEI CERT C++ Coding Standard, OWASP Top 10
* **Core Concepts:** Zero Trust, Defense in Depth, CI/CD Pipeline Automation, Principle of Least Privilege, Static Analysis.
